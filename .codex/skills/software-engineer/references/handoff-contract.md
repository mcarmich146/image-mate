# Requirement Handoff Contract

Use this checklist when receiving requirements from requirement-engineer before implementation.

## 1. Problem and Outcome

- Problem statement is explicit.
- Business/user outcome is measurable.
- Success metric and target threshold are defined.

## 2. Scope Boundaries

- In-scope behaviors are listed.
- Out-of-scope behaviors are listed.
- Assumptions and dependencies are named.

## 3. Functional Requirements

- Each requirement has a stable ID.
- Input, behavior, and expected output are explicit.
- Error and validation behavior are defined.

## 4. Non-Functional Requirements

- Performance constraints are measurable (latency, throughput, resource limits).
- Reliability and availability expectations are explicit.
- Security/privacy constraints are explicit.
- Compatibility and migration constraints are explicit.

## 5. Acceptance Criteria

- Criteria are testable and deterministic.
- Edge cases are included.
- Negative/failure paths are included.
- Done condition is clear enough for independent verification.

## 6. Delivery Constraints

- Technology or platform constraints are listed.
- Timeline, sequencing, or release constraints are listed.
- Required documentation and operational readiness artifacts are listed.

## 7. Open Questions Gate

If any section above is incomplete, record blockers and resolve before coding starts.
